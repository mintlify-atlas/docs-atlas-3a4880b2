---
title: 'FAQ'
description: 'Answers to common questions about WP Manager Pro.'
---

<AccordionGroup>

  <Accordion title="How do I open WP Manager Pro?">
    After activating the plugin, click **WP Manager** in the WordPress admin sidebar. You can also click the **Open** link in the Plugins list row next to WP Manager Pro.
  </Accordion>

  <Accordion title="Who can access WP Manager Pro?">
    Only WordPress users with the `manage_options` capability — Administrators. Every REST API endpoint enforces this check. Lower-privileged users (Editors, Authors, etc.) receive a `403 Forbidden` response and cannot access any part of the interface.
  </Accordion>

  <Accordion title="How do I put my site in maintenance mode?">
    Go to **Maintenance** in the sidebar.

    - Use the **Content** tab to set your title and message.
    - Use the **Appearance** tab to choose a gradient preset and icon.
    - Use the **Extras** tab to add a countdown timer or status badge.

    Click **Enable Maintenance** to go live, or **Save Settings** to persist your design without changing the active state. The maintenance page is removed automatically when you disable it or deactivate the plugin.
  </Accordion>

  <Accordion title="How do I downgrade a plugin or theme to an older version?">
    In **Plugin Manager** or **Theme Manager**, click the version history icon on any plugin or theme row. A dialog loads all available versions from WordPress.org. Click **Install** next to the version you want — the plugin or theme stays active after the switch.
  </Accordion>

  <Accordion title="How do I hide the WordPress login page?">
    Go to **Security → Login** tab. Enter a secret slug (for example, `my-secret-login`) and click **Enable Protection**. Your login page moves to `yoursite.com/my-secret-login`. Direct GET requests to `wp-login.php` are redirected to the homepage.

    Copy the new URL using the clipboard button before saving. To restore the default login URL, click **Disable Protection**.
  </Accordion>

  <Accordion title="How do I block brute-force login attacks?">
    Go to **Security → Login** tab. Enable the **Login Attempt Limiter** and configure:

    - **Max attempts** — how many failed attempts are allowed before a lockout
    - **Counting window** — the time window in seconds over which attempts are counted
    - **Lockout duration** — how long in seconds the IP is blocked

    When an IP exceeds the limit it is automatically blocked. View and unlock locked-out IPs in the **Lockout Log** section on the same tab.
  </Accordion>

  <Accordion title="How do I block a specific IP or IP range?">
    Go to **Security → Hardening** tab. Enter an IP address (for example, `203.0.113.5`) or a CIDR range (for example, `10.0.0.0/24`) in the IP Blocklist field, add an optional note, and click **Add**.

    Blocked IPs are rejected on WordPress's `init` hook before any page output is sent.
  </Accordion>

  <Accordion title="How do I verify my WordPress core files haven't been tampered with?">
    Go to **Security → Integrity** tab and click **Run Integrity Check**. WP Manager Pro fetches the official MD5 checksums from wordpress.org and compares every file in `wp-admin/` and `wp-includes/`. Any modified or missing files are listed with their actual hash, expected hash, and last-modified date.
  </Accordion>

  <Accordion title="How do I enable Two-Factor Authentication?">
    Go to **Security → Two-Factor** tab and click **Set Up Two-Factor Auth**. Scan the QR code with Google Authenticator, Authy, or any TOTP app, then enter the 6-digit code to verify and activate.

    Eight one-time backup codes are generated and shown once — save them somewhere safe. To disable 2FA, click **Disable 2FA** on the same tab.
  </Accordion>

  <Accordion title="How do I enable WebP or AVIF image uploads?">
    Go to **Image Tools** and toggle **WebP Conversion** or **AVIF Support** on.

    - WebP requires GD or ImageMagick.
    - AVIF requires PHP 8.1+ with GD (`imageavif` function) or ImageMagick compiled with the AVIF codec.

    The support status cards at the top of the Image Tools page show exactly what your server supports.
  </Accordion>

  <Accordion title="How do I use Login As (admin impersonation)?">
    Go to **Users**, find the user you want to impersonate, and click the Login As icon on their row. WP Manager Pro generates a secure single-use token (valid for 5 minutes) and redirects you to WP Admin logged in as that user.

    Only existing administrators can trigger this. The token is validated server-side and deleted immediately on first use.
  </Accordion>

  <Accordion title="Is it safe to edit files with the File Manager?">
    Yes — all file paths are validated against `ABSPATH` using `realpath()` to prevent path traversal attacks. `wp-config.php`, `.htaccess`, and `index.php` are write-protected from deletion and rename. The editor is limited to text-based file types with a 2 MB read limit.

    That said, treat the File Manager like any code editor: incorrect edits to PHP files can break your site. Always keep a backup.
  </Accordion>

  <Accordion title="Can I run custom SQL queries in the Database Manager?">
    The **SQL Runner** tab is read-only. It accepts `SELECT`, `SHOW`, `DESCRIBE`, and `EXPLAIN` statements only. Write operations (`INSERT`, `UPDATE`, `DELETE`, `DROP`, and others) are blocked at the API level.

    For write operations, use the **Table Data** tab to insert, edit, or delete individual rows through the UI.
  </Accordion>

  <Accordion title="How do I check for plugin and theme updates?">
    Click the **Check Updates** button in the top-right area of Plugin Manager or Theme Manager. This forces WordPress to re-query WordPress.org for fresh update data and immediately refreshes the list — no page reload needed.
  </Accordion>

  <Accordion title="Does WP Manager Pro support WordPress Multisite?">
    The plugin works on Multisite but is designed for single-site use. It installs and activates on a per-site basis. Network-level management (network activation, super-admin actions) is not currently supported.
  </Accordion>

  <Accordion title="How do I uninstall WP Manager Pro?">
    Deactivate the plugin from **WP Admin → Plugins**. On deactivation, the maintenance file (`wp-content/maintenance.php`) is automatically removed.

    Deactivating does **not** delete your notes or settings stored in `wp_options`. These are cleaned up if you manually delete the plugin afterward.
  </Accordion>

  <Accordion title="Where are notes stored?">
    Notes are stored in a custom `wp_wmp_notes` database table created on plugin activation. They persist across plugin updates and are not affected by deactivation.
  </Accordion>

  <Accordion title="Why does the page look unstyled or show a blank white area?">
    This usually means another plugin or theme is loading a conflicting script on the WP Manager Pro page. Check the browser console for JavaScript errors.

    If you see a Content Security Policy (CSP) violation, your server's CSP headers may be blocking the inline React app. You may need to whitelist the plugin's script source in your CSP configuration.
  </Accordion>

</AccordionGroup>

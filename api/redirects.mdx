---
title: 'Redirects'
description: 'Manage 301/302/307 URL redirects with CSV import and export support.'
---

<Info>
  All endpoints require a valid WordPress nonce in the `X-WP-Nonce` header and the `manage_options` capability. Added in v1.4.0.
</Info>

<Note>
  Redirects are stored in the `wp_wmp_redirects` table and are enforced on `template_redirect`. Source paths are automatically normalized to begin with `/`. Wildcard `*` patterns in the source are not matched server-side — they are stored as-is.
</Note>

**Base URL:** `{site_url}/wp-json/wp-manager-pro/v1`

---

## GET /redirects

Returns a paginated list of redirects.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/redirects?page=1&per_page=25" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Parameters

<ParamField query="page" type="number" default="1">
  Page number.
</ParamField>

<ParamField query="per_page" type="number" default="50">
  Results per page (10–200).
</ParamField>

<ParamField query="search" type="string">
  Filter by source or target URL fragment.
</ParamField>

### Response

<ResponseField name="redirects" type="array">
  Array of redirect objects, each with `id`, `source`, `target`, `type`, `hits`, `created_at`, and `updated_at`.
</ResponseField>

<ResponseField name="total" type="number">
  Total matching redirects.
</ResponseField>

<ResponseField name="page" type="number">
  Current page.
</ResponseField>

<ResponseField name="per_page" type="number">
  Results per page.
</ResponseField>

<ResponseField name="total_pages" type="number">
  Total pages.
</ResponseField>

## POST /redirects

Create a new redirect rule.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/redirects" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"source": "/old-page", "target": "/new-page", "type": 301}'
```

### Parameters

<ParamField body="source" type="string" required>
  Source URL path (automatically prepended with `/` if missing). Must be unique.
</ParamField>

<ParamField body="target" type="string" required>
  Destination URL or path.
</ParamField>

<ParamField body="type" type="number" default="301">
  HTTP redirect status code. Accepted values: `301`, `302`, `307`.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the redirect was created.
</ResponseField>

<ResponseField name="redirect" type="object">
  The newly created redirect object.
</ResponseField>

## PUT /redirects/{id}

Update an existing redirect rule.

```bash
curl -X PUT "https://yoursite.com/wp-json/wp-manager-pro/v1/redirects/5" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"target": "/updated-destination", "type": 302}'
```

### Path parameters

<ParamField path="id" type="number" required>
  The redirect ID to update.
</ParamField>

### Body parameters

Accepts `source`, `target`, and `type`. Fields omitted from the request retain their existing values.

### Response

<ResponseField name="success" type="boolean">
  Whether the redirect was updated.
</ResponseField>

<ResponseField name="redirect" type="object">
  The updated redirect object.
</ResponseField>

### Response

<ResponseField name="success" type="boolean">

## POST /redirects/export

Export all redirects as CSV. Returns a temporary download URL valid for 120 seconds.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/redirects/export" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">
  Whether the export was prepared.
</ResponseField>

<ResponseField name="download_url" type="string">
  One-time URL to download the CSV file (expires in 120 seconds).
</ResponseField>

---

## GET /redirects/download

Stream the prepared CSV file. This URL is generated by `POST /redirects/export` and is consumed once.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/redirects/download?key=KEY&_wpnonce=NONCE" \
  --output redirects.csv
```

### Parameters

<ParamField query="key" type="string" required>
  The one-time download key returned by `/redirects/export`.
</ParamField>

### Response

Streams a `text/csv` file with headers `ID`, `Source`, `Target`, `Type`, `Hits`, and `Created At`.

---

## POST /redirects/import

Import redirects from a CSV file. Duplicate source paths are skipped silently. Invalid redirect types default to `301`.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/redirects/import" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -F "file=@redirects.csv"
```

### Parameters

<ParamField body="file" type="file" required>
  CSV file upload. Expected columns (in order): `source`, `target`, `type`. A header row is detected and skipped automatically.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the import completed.
</ResponseField>

<ResponseField name="imported" type="number">
  Number of redirects successfully imported.
</ResponseField>

---
title: 'Plugins'
description: 'List, activate, deactivate, delete, install, search, upload, export, update, and manage plugin versions through the REST API.'
---

<Note>
All plugin endpoints require the `manage_options` capability. WP Manager Pro cannot deactivate or delete itself.
</Note>

---

## GET /plugins

Returns all installed plugins with their activation status and pending update flag. Results are sorted alphabetically by name.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response fields

<ResponseField name="plugins" type="object[]" required>
  Array of plugin objects.
  <Expandable title="item properties">
    <ResponseField name="file" type="string">Plugin file path relative to the plugins directory, e.g. `"akismet/akismet.php"`.</ResponseField>
    <ResponseField name="name" type="string">Plugin display name.</ResponseField>
    <ResponseField name="version" type="string">Installed version.</ResponseField>
    <ResponseField name="description" type="string">Plugin description.</ResponseField>
    <ResponseField name="author" type="string">Author name (may contain HTML).</ResponseField>
    <ResponseField name="author_uri" type="string">Author URL.</ResponseField>
    <ResponseField name="plugin_uri" type="string">Plugin homepage URL.</ResponseField>
    <ResponseField name="text_domain" type="string">Text domain for translations.</ResponseField>
    <ResponseField name="active" type="boolean">`true` if the plugin is currently active.</ResponseField>
    <ResponseField name="has_update" type="boolean">`true` if a newer version is available on WordPress.org.</ResponseField>
    <ResponseField name="network" type="boolean">`true` if this is a network-only plugin.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number" required>
  Total number of installed plugins.
</ResponseField>

## POST /plugins/activate

Activates an installed plugin.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/activate" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"plugin": "akismet/akismet.php"}'
```

### Request parameters

<ParamField body="plugin" type="string" required>
  Plugin file path relative to the plugins directory, e.g. `"akismet/akismet.php"`.
</ParamField>

---

## POST /plugins/deactivate

Deactivates an active plugin. Returns `403` if you attempt to deactivate WP Manager Pro itself.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/deactivate" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"plugin": "akismet/akismet.php"}'
```

### Request parameters

<ParamField body="plugin" type="string" required>
  Plugin file path relative to the plugins directory.
</ParamField>

---

## DELETE /plugins/delete

Deactivates (if active) and permanently deletes a plugin. Returns `403` if you attempt to delete WP Manager Pro itself.

```bash
curl -X DELETE "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/delete" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"plugin": "hello-dolly/hello.php"}'
```

### Request parameters

<ParamField body="plugin" type="string" required>
  Plugin file path relative to the plugins directory.
</ParamField>

## POST /plugins/install

Installs a plugin from the WordPress.org repository by slug.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/install" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "query-monitor"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  WordPress.org plugin slug, e.g. `"query-monitor"`.
</ParamField>

## GET /plugins/search

Searches the WordPress.org plugin repository. Returns up to 12 results per page.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/search?q=seo&page=1" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Request parameters

<ParamField query="q" type="string" required>
  Search query string.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

### Response fields

<ResponseField name="plugins" type="object[]">
  Search results from WordPress.org.
  <Expandable title="item properties">
    <ResponseField name="slug" type="string">Plugin slug.</ResponseField>
    <ResponseField name="name" type="string">Plugin name.</ResponseField>
    <ResponseField name="version" type="string">Latest version.</ResponseField>
    <ResponseField name="short_description" type="string">Short plugin description.</ResponseField>
    <ResponseField name="author" type="string">Author name.</ResponseField>
    <ResponseField name="rating" type="number">Rating out of 100.</ResponseField>
    <ResponseField name="num_ratings" type="number">Total number of ratings.</ResponseField>
    <ResponseField name="downloaded" type="number">Total download count.</ResponseField>
    <ResponseField name="last_updated" type="string">Last updated date string.</ResponseField>
    <ResponseField name="icon" type="string">URL to the plugin icon image.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">Total number of results matching the search.</ResponseField>
<ResponseField name="pages" type="number">Total number of result pages.</ResponseField>

## POST /plugins/upload

Installs a plugin from an uploaded `.zip` file. Only `.zip` files are accepted.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/upload" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -F "file=@/path/to/plugin.zip" \
  -F "overwrite=1"
```

### Request parameters

<ParamField body="file" type="file" required>
  The plugin `.zip` file to upload (multipart/form-data).
</ParamField>

<ParamField body="overwrite" type="boolean" default="false">
  Set to `1` or `true` to overwrite an existing plugin with the same slug.
</ParamField>

## POST /plugins/export

Creates a `.zip` archive of an installed plugin and returns a temporary download URL. The download URL is valid for 5 minutes.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/export" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"plugin": "my-plugin/my-plugin.php"}'
```

### Request parameters

<ParamField body="plugin" type="string" required>
  Plugin file path relative to the plugins directory.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">Always `true` on success.</ResponseField>
<ResponseField name="download_url" type="string">Authenticated URL to stream the ZIP file download. Valid for 5 minutes.</ResponseField>
<ResponseField name="filename" type="string">Suggested filename for the download, e.g. `"my-plugin.zip"`.</ResponseField>

---

## GET /plugins/download

Streams a previously prepared plugin ZIP export to the browser. This is called via the `download_url` returned by `POST /plugins/export`.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/download?key=KEY&_wpnonce=NONCE" \
  --output plugin.zip
```

### Request parameters

<ParamField query="key" type="string" required>
  The temporary download key returned by `POST /plugins/export`.
</ParamField>

<Warning>
The download key is single-use and expires after 5 minutes. The ZIP file is deleted from the server after being streamed.
</Warning>

## POST /plugins/update

Updates an installed plugin to the latest version available on WordPress.org. If the plugin was active before the update, it is re-activated automatically.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/update" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"plugin": "akismet/akismet.php"}'
```

### Request parameters

<ParamField body="plugin" type="string" required>
  Plugin file path relative to the plugins directory. Returns `400` if no update is available for this plugin.
</ParamField>

## POST /plugins/install-version

Installs a specific version of a plugin from WordPress.org. Useful for downgrading or pinning to a known-good version. If the plugin was active before the install, it is re-activated automatically.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/plugins/install-version" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "akismet", "version": "5.2"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  WordPress.org plugin slug. Must match the pattern `[a-z0-9-]+`.
</ParamField>

<ParamField body="version" type="string" required>
  Version string to install, e.g. `"5.2"`. Must match the pattern `[a-zA-Z0-9.-]+`.
</ParamField>

### Common success response

All mutating plugin endpoints return the same success shape:

```json
{
  "success": true,
  "message": "Plugin activated successfully."
}
```

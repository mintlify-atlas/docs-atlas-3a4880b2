---
title: 'Audit Log'
description: 'Query, export, and clear the audit log that automatically tracks plugin, theme, user, and content events.'
---

<Info>
  All endpoints require a valid WordPress nonce in the `X-WP-Nonce` header and the `manage_options` capability. Added in v1.4.0.
</Info>

<Note>
  The audit log is stored in `wp_wmp_audit_log`. Events are recorded automatically for plugin activation/deactivation/deletion, theme activation, user login/logout/registration/login failures, post publishing, and manual WP Manager Pro operations.
</Note>

**Base URL:** `{site_url}/wp-json/wp-manager-pro/v1`

---

## GET /audit

Returns paginated audit log entries, newest first. Supports filtering by action type, user, and date range.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/audit?page=1&per_page=25&action_type=plugin.activated" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Parameters

<ParamField query="page" type="number" default="1">
  Page number.
</ParamField>

<ParamField query="per_page" type="number" default="50">
  Results per page (10–200).
</ParamField>

<ParamField query="action_type" type="string">
  Filter to a specific action. Use `GET /audit/action-types` for available values.
</ParamField>

<ParamField query="user_id" type="number">
  Filter to a specific WordPress user ID.
</ParamField>

<ParamField query="from" type="string">
  Start date filter in `YYYY-MM-DD` format (inclusive, from midnight).
</ParamField>

<ParamField query="to" type="string">
  End date filter in `YYYY-MM-DD` format (inclusive, through 23:59:59).
</ParamField>

### Response

<ResponseField name="logs" type="array">
  <Expandable title="log entry properties">
    <ResponseField name="id" type="number">Log entry ID.</ResponseField>
    <ResponseField name="user_id" type="number">WordPress user ID.</ResponseField>
    <ResponseField name="user_name" type="string">WordPress username.</ResponseField>
    <ResponseField name="action" type="string">Event action (e.g. `plugin.activated`).</ResponseField>
    <ResponseField name="object_type" type="string">Type of affected object (e.g. `plugin`, `user`, `post`).</ResponseField>
    <ResponseField name="object_name" type="string">Name or identifier of the affected object.</ResponseField>
    <ResponseField name="extra" type="string | null">JSON-encoded extra context, or `null`.</ResponseField>
    <ResponseField name="ip_address" type="string">Client IP address at the time of the event.</ResponseField>
    <ResponseField name="created_at" type="string">Event timestamp (`Y-m-d H:i:s`).</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Total matching log entries.
</ResponseField>

<ResponseField name="page" type="number">
  Current page.
</ResponseField>

<ResponseField name="per_page" type="number">
  Results per page.
</ResponseField>

<ResponseField name="total_pages" type="number">
  Total pages.
</ResponseField>

## DELETE /audit/clear

Truncate the audit log table. A single `audit.cleared` event is written after the truncation.

```bash
curl -X DELETE "https://yoursite.com/wp-json/wp-manager-pro/v1/audit/clear" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">
  Whether the log was cleared.
</ResponseField>

---

## POST /audit/export

Export the full audit log as a CSV and return a temporary download URL.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/audit/export" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="success" type="boolean">
  Whether the export was prepared.
</ResponseField>

<ResponseField name="download_url" type="string">
  One-time download URL valid for 120 seconds.
</ResponseField>

---

## GET /audit/download

Stream the prepared CSV file. The URL is generated by `POST /audit/export` and is consumed once.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/audit/download?key=KEY&_wpnonce=NONCE" \
  --output audit-log.csv
```

### Parameters

<ParamField query="key" type="string" required>
  The one-time export key returned by `POST /audit/export`.
</ParamField>

### Response

Streams a `text/csv` file with columns: `ID`, `User ID`, `User`, `Action`, `Object Type`, `Object Name`, `Extra`, `IP Address`, `Date`.

---

## GET /audit/action-types

Returns a list of distinct action type strings present in the log, for use as filter values.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/audit/action-types" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="types" type="array">
  Alphabetically sorted array of action type strings (e.g. `["backup.created", "plugin.activated", "user.login"]`).
</ResponseField>

---
title: 'Backup'
description: 'Create, download, and delete SQL database backups. Configure automatic scheduled backups via WP-Cron.'
---

<Info>
  All endpoints require a valid WordPress nonce in the `X-WP-Nonce` header and the `manage_options` capability. Added in v1.4.0; scheduled backups added in v1.8.0.
</Info>

<Note>
  Backups are stored as `.sql` files in `wp-content/wmp-backups/`, which is protected from direct web access by an `.htaccess` file.
</Note>

**Base URL:** `{site_url}/wp-json/wp-manager-pro/v1`

---

## GET /backup

Returns a list of available backup files sorted newest first.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/backup" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="backups" type="array">
  <Expandable title="backup properties">
    <ResponseField name="name" type="string">Filename (e.g. `backup-2024-01-15-103045.sql`).</ResponseField>
    <ResponseField name="size" type="number">File size in bytes.</ResponseField>
    <ResponseField name="size_human" type="string">Human-readable file size.</ResponseField>
    <ResponseField name="created_at" type="string">File modification date (`Y-m-d H:i:s`).</ResponseField>
  </Expandable>
</ResponseField>

## POST /backup/create

Create a new SQL dump. Omit `tables` for a full database backup.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/create" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{}'
```

```bash
# Specific tables
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/create" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"tables": ["wp_posts", "wp_postmeta"]}'
```

### Parameters

<ParamField body="tables" type="string[]">
  Array of table names to include. Omit or pass `null` for a full database backup.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the backup was created.
</ResponseField>

<ResponseField name="name" type="string">
  The backup filename.
</ResponseField>

<ResponseField name="size" type="number">
  File size in bytes.
</ResponseField>

<ResponseField name="size_human" type="string">
  Human-readable file size.
</ResponseField>

<ResponseField name="created_at" type="string">
  Backup creation timestamp.
</ResponseField>

## POST /backup/download

Prepare a backup file for download. Returns a temporary URL valid for 120 seconds.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/download" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"name": "backup-2024-01-15-103045.sql"}'
```

### Parameters

<ParamField body="name" type="string" required>
  The backup filename returned by `GET /backup` or `POST /backup/create`. Must match the pattern `backup-YYYYMMDD-HHmmss.sql`.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the download URL was prepared.
</ResponseField>

<ResponseField name="download_url" type="string">
  One-time URL to stream the backup file (expires in 120 seconds).
</ResponseField>

## GET /backup/serve

Stream a backup file as an attachment download. The URL is generated by `POST /backup/download` and is consumed once.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/serve?key=KEY&_wpnonce=NONCE" \
  --output backup.sql
```

### Parameters

<ParamField query="key" type="string" required>
  The one-time download key from `POST /backup/download`.
</ParamField>

### Response

Streams the `.sql` file with `Content-Type: application/sql`.

## DELETE /backup/delete

Permanently delete a backup file.

```bash
curl -X DELETE "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/delete" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"name": "backup-2024-01-15-103045.sql"}'
```

### Parameters

<ParamField body="name" type="string" required>
  The backup filename to delete.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the file was deleted.
</ResponseField>

## GET /backup/schedule

Returns the current scheduled backup configuration.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/schedule" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response

<ResponseField name="enabled" type="boolean">
  Whether scheduled backups are active.
</ResponseField>

<ResponseField name="frequency" type="string">
  Backup frequency: `daily`, `weekly`, or `monthly`.
</ResponseField>

<ResponseField name="retain" type="number">
  Maximum number of backup files to keep. Oldest files are pruned after each run.
</ResponseField>

<ResponseField name="next_run" type="string | null">
  Next scheduled run timestamp (`Y-m-d H:i:s`), or `null` if scheduling is disabled.
</ResponseField>

---

## POST /backup/schedule

Save scheduled backup configuration and register or clear the WP-Cron event.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/backup/schedule" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"enabled": true, "frequency": "daily", "retain": 7}'
```

### Parameters

<ParamField body="enabled" type="boolean" required>
  Enable or disable the scheduled backup cron.
</ParamField>

<ParamField body="frequency" type="string" default="daily">
  How often to run. Accepted values: `daily`, `weekly`, `monthly`.
</ParamField>

<ParamField body="retain" type="number" default="5">
  Number of most recent backups to keep. Older files are deleted automatically.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the schedule was saved.
</ResponseField>

<ResponseField name="enabled" type="boolean">
  New enabled state.
</ResponseField>

<ResponseField name="frequency" type="string">
  The configured frequency.
</ResponseField>

<ResponseField name="retain" type="number">
  The configured retain count.
</ResponseField>

<ResponseField name="next_run" type="string | null">
  Next scheduled run timestamp, or `null` if disabled.
</ResponseField>

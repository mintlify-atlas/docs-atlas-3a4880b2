---
title: 'Themes'
description: 'List, activate, delete, install, search, upload, export, update, and manage theme versions through the REST API.'
---

<Note>
All theme endpoints require the `manage_options` capability. The active theme cannot be deleted.
</Note>

---

## GET /themes

Returns all installed themes. The active theme is sorted first; remaining themes are sorted alphabetically.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/themes" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Response fields

<ResponseField name="themes" type="object[]" required>
  Array of theme objects.
  <Expandable title="item properties">
    <ResponseField name="slug" type="string">Theme directory slug, e.g. `"twentytwentyfour"`.</ResponseField>
    <ResponseField name="name" type="string">Theme display name.</ResponseField>
    <ResponseField name="version" type="string">Installed version.</ResponseField>
    <ResponseField name="description" type="string">Theme description.</ResponseField>
    <ResponseField name="author" type="string">Author name.</ResponseField>
    <ResponseField name="author_uri" type="string">Author URL.</ResponseField>
    <ResponseField name="theme_uri" type="string">Theme homepage URL.</ResponseField>
    <ResponseField name="screenshot" type="string">Absolute URL to the theme screenshot image, or empty string if none.</ResponseField>
    <ResponseField name="active" type="boolean">`true` if this is the currently active theme.</ResponseField>
    <ResponseField name="is_child" type="boolean">`true` if this theme is a child theme.</ResponseField>
    <ResponseField name="parent" type="string | null">Parent theme display name if this is a child theme, otherwise `null`.</ResponseField>
    <ResponseField name="has_update" type="boolean">`true` if a newer version is available on WordPress.org.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number" required>
  Total number of installed themes.
</ResponseField>

## POST /themes/activate

Activates an installed theme.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/activate" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "twentytwentyfour"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  Theme directory slug.
</ParamField>

---

## DELETE /themes/delete

Permanently deletes an inactive theme. Returns `403` if you attempt to delete the currently active theme.

```bash
curl -X DELETE "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/delete" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "twentytwentyone"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  Theme directory slug of the theme to delete.
</ParamField>

## POST /themes/install

Installs a theme from the WordPress.org repository by slug.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/install" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "astra"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  WordPress.org theme slug.
</ParamField>

## GET /themes/search

Searches the WordPress.org theme repository. Returns up to 12 results per page.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/search?q=blog&page=1" \
  -H "X-WP-Nonce: YOUR_NONCE"
```

### Request parameters

<ParamField query="q" type="string" required>
  Search query string.
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for paginated results.
</ParamField>

### Response fields

<ResponseField name="themes" type="object[]">
  Search results from WordPress.org.
  <Expandable title="item properties">
    <ResponseField name="slug" type="string">Theme slug.</ResponseField>
    <ResponseField name="name" type="string">Theme name.</ResponseField>
    <ResponseField name="version" type="string">Latest version.</ResponseField>
    <ResponseField name="description" type="string">Theme description.</ResponseField>
    <ResponseField name="author" type="string">Author username.</ResponseField>
    <ResponseField name="screenshot_url" type="string">URL to the theme screenshot.</ResponseField>
    <ResponseField name="rating" type="number">Rating out of 100.</ResponseField>
    <ResponseField name="num_ratings" type="number">Total number of ratings.</ResponseField>
    <ResponseField name="downloaded" type="number">Total download count.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">Total results matching the search.</ResponseField>

## POST /themes/upload

Installs a theme from an uploaded `.zip` file. Only `.zip` files are accepted.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/upload" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -F "file=@/path/to/theme.zip" \
  -F "overwrite=1"
```

### Request parameters

<ParamField body="file" type="file" required>
  The theme `.zip` file to upload (multipart/form-data).
</ParamField>

<ParamField body="overwrite" type="boolean" default="false">
  Set to `1` or `true` to overwrite an existing theme with the same slug.
</ParamField>

## POST /themes/export

Creates a `.zip` archive of an installed theme and returns a temporary download URL. The download URL is valid for 5 minutes.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/export" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "my-theme"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  Theme directory slug.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">Always `true` on success.</ResponseField>
<ResponseField name="download_url" type="string">Authenticated URL to stream the ZIP file download. Valid for 5 minutes.</ResponseField>
<ResponseField name="filename" type="string">Suggested filename, e.g. `"my-theme.zip"`.</ResponseField>

---

## GET /themes/download

Streams a previously prepared theme ZIP export. Called via the `download_url` returned by `POST /themes/export`.

```bash
curl -X GET "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/download?key=KEY&_wpnonce=NONCE" \
  --output theme.zip
```

### Request parameters

<ParamField query="key" type="string" required>
  The temporary download key returned by `POST /themes/export`.
</ParamField>

<Warning>
The download key is single-use and expires after 5 minutes. The ZIP file is deleted from the server after being streamed.
</Warning>

## POST /themes/update

Updates an installed theme to its latest version available on WordPress.org.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/update" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "astra"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  Theme directory slug. Returns `400` if no update is available.
</ParamField>

## POST /themes/install-version

Installs a specific version of a theme from WordPress.org. Useful for downgrading or pinning to a known-good version.

```bash
curl -X POST "https://yoursite.com/wp-json/wp-manager-pro/v1/themes/install-version" \
  -H "X-WP-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{"slug": "astra", "version": "4.5.0"}'
```

### Request parameters

<ParamField body="slug" type="string" required>
  WordPress.org theme slug. Must match the pattern `[a-z0-9-]+`.
</ParamField>

<ParamField body="version" type="string" required>
  Version string to install, e.g. `"4.5.0"`. Must match the pattern `[a-zA-Z0-9.-]+`.
</ParamField>

### Common success response

All mutating theme endpoints return the same success shape:

```json
{
  "success": true,
  "message": "Theme activated successfully."
}
```

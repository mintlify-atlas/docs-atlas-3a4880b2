**WP Manager Pro** replaces the need for multiple separate admin plugins by providing one fast, modern interface for managing plugins, themes, files, databases, users, security, and more. Install it as a standard WordPress plugin and a full React-powered control panel appears under your WP Admin menu.

All operations run through a secured REST API namespace (`wp-manager-pro/v1`). Every endpoint requires the `manage_options` WordPress capability, so only site administrators can access the dashboard.

## Why WP Manager Pro?

Managing a WordPress site typically means juggling many separate plugins — one for backups, one for security, one for file editing, one for redirects. WP Manager Pro consolidates all of that into a single, cohesive interface that loads fast and stays out of your way.

For agencies managing multiple client sites, the consolidated toolset reduces onboarding friction: every site gets the same familiar dashboard, the same keyboard shortcuts, and the same REST API surface.

## Architecture overview

WP Manager Pro is split into two layers that communicate entirely through the WordPress REST API.

**Frontend — React application**

The frontend is a single-page React 19 application written in TypeScript. It is compiled by Vite and served as two static files (`assets/build/index.js` and `assets/build/style.css`) that WordPress enqueues on the plugin's admin page. React Router (HashRouter) handles client-side navigation between sections. TanStack Query manages server state and caching. The Monaco Editor (VS Code engine) is embedded inside the File Manager and Code Snippets editor.

When a user navigates to **WP Admin → WP Manager**, WordPress renders a single `<div id="wp-manager-pro-root">` element. `main.tsx` mounts the React app into that container. From that point on, all navigation and data fetching happens within the React app — no full page reloads.

**Backend — PHP REST API**

The backend is a standard WordPress plugin written in PHP 7.4+. A singleton `Plugin` class bootstraps all hooks and loads 27 REST API controllers. Every controller registers its routes under the `wp-manager-pro/v1` namespace via `rest_api_init`. All routes use WordPress nonce authentication (`wp_rest`) and enforce the `manage_options` capability.

```
Browser
  └── React app (HashRouter)
        └── TanStack Query fetch
              └── X-WP-Nonce header
                    └── wp-json/wp-manager-pro/v1/...
                          └── PHP controller
                                └── WordPress core functions
```
